About SQLi Playground

This web app demonstrates how SQL Injection vulnerabilities work and how to prevent them using parameterized queries.

In this version, user inputs are safely passed as parameters, preventing malicious SQL code from executing.

Along with demonstrating vulnerabilities, this playground also includes a SQL Injection Detector.

The detector compares user inputs against a list of known malicious patterns, such as common SQL injection keywords and operators.

- If suspicious input is found, it is flagged as a possible SQL injection attempt.
- Each attempt is logged with details and timestamps for analysis.
- If no match is found, the input is considered safe.

This shows how detection systems can help identify malicious activity, but also highlights that real-world systems use more advanced techniques than this simple educational demo.
